## v0.1.0 (2026-07-03)

Initial release. cairn is an artifact-native pipeline orchestrator: every step leaves a validated
artifact on disk, the trail of artifacts *is* the execution state, and resume means walking that
trail to the last valid cairn.

### Features

- **Kernel** — declarative pipelines over a planner, walker, gate engine, artifact composer, trail
  and run-state, guards, and the expression + template engines, with the filesystem as the only
  state.
- **Executors** — five step executors behind one protocol: `shell` and `stub` (the offline test
  executor), plus the **`claude`, `codex`, and `grok`** coding-agent CLIs, each live-verified and a
  **mixed fleet** proven in a single pipeline (codex → claude → grok) with per-step models recorded
  in `run.json`.
- **CLI** — the full verb set is live: `plan`, `run`, `resume`, `gate`, `trail`, `doctor`,
  `new`, `test`, and the `batch`, `learnings`, `gc`, and `schedule` verbs.
- **Scheduling** — first-class `schedules.yaml` with cron / launchd / systemd installers and
  content-key idempotency.
- **Observability** — opt-in `heartbeat` trail events, a webhook trail sink (a tee, never the
  authority), and kernel-side redaction of declared secrets from step logs and the trail.
- **Safety & compatibility** — cross-version resume gates, range-scoped tool enforcement at plan
  and run time, and a single aware-UTC clock behind the kernel.
- **Workspace test layer** — `cairn test` walks a workspace's validators, guards, pipelines, and
  envelopes through the stub executor (with `record` for offline replays).
- **Scaffold** — `cairn new workspace` instantiates a runnable workspace whose `hello` pipeline
  runs offline with zero auth, and ships the self-improve pipeline (the learning loop's
  curate → promote as packaged workspace furniture).
